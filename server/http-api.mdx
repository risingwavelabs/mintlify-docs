Wavelet exposes a REST API for reading queries, writing events, and checking server health. All responses are JSON. CORS is enabled by default (all origins).

Base URL: `http://localhost:8080` (configurable via `server.port` and `server.host` in your config).

## Endpoints

### Health check

```
GET /v1/health
```

```json
{ "status": "ok" }
```

### List queries

```
GET /v1/queries
```

Returns all available query names.

```json
{ "queries": ["leaderboard", "model_stats"] }
```

### List events

```
GET /v1/events
```

Returns all available event names.

```json
{ "events": ["game_events", "llm_events"] }
```

### Query data

```
GET /v1/queries/{name}
```

Returns the current rows of a materialized view.

```bash
curl http://localhost:8080/v1/queries/leaderboard
```

```json
{
  "query": "leaderboard",
  "rows": [
    { "player_id": "alice", "total_score": 350 },
    { "player_id": "bob", "total_score": 200 }
  ]
}
```

**Query parameter filtering:** Add query parameters to filter rows by column values.

```bash
curl "http://localhost:8080/v1/queries/leaderboard?player_id=alice"
```

```json
{
  "query": "leaderboard",
  "rows": [
    { "player_id": "alice", "total_score": 350 }
  ]
}
```

Multiple parameters are combined with `AND`:

```bash
curl "http://localhost:8080/v1/queries/orders?status=pending&region=us-east"
```

**Error responses:**

| Status | Body | When |
|--------|------|------|
| `404` | `{ "error": "Query 'foo' not found.", "available_queries": ["leaderboard"] }` | Query does not exist |
| `401` | `{ "error": "Authentication required..." }` | JWT configured but no valid token |

### Write a single event

```
POST /v1/events/{name}
```

Insert one row into an event table.

```bash
curl -X POST http://localhost:8080/v1/events/game_events \
  -H "Content-Type: application/json" \
  -d '{"player_id": "alice", "action": "win", "score": 100}'
```

```json
{ "ok": true }
```

**Error responses:**

| Status | Body | When |
|--------|------|------|
| `404` | `{ "error": "Event 'foo' not found.", "available_events": ["game_events"] }` | Event does not exist |
| `400` | `{ "error": "..." }` | Invalid data or insert failure |

### Write a batch of events

```
POST /v1/events/{name}/batch
```

Insert multiple rows in one request. Body must be a JSON array.

```bash
curl -X POST http://localhost:8080/v1/events/game_events/batch \
  -H "Content-Type: application/json" \
  -d '[
    {"player_id": "alice", "action": "win", "score": 100},
    {"player_id": "bob", "action": "loss", "score": -25}
  ]'
```

```json
{ "ok": true }
```

## Authentication

When JWT is configured, pass the token via the `Authorization` header:

```bash
curl http://localhost:8080/v1/queries/leaderboard \
  -H "Authorization: Bearer eyJhbG..."
```

JWT is required for WebSocket subscriptions when configured, but the HTTP API currently accepts requests without a token for query reads and event writes. JWT filtering only applies to WebSocket diffs.

## CORS

All endpoints return these headers:

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: Content-Type, Authorization
```

`OPTIONS` requests return `204 No Content`.
