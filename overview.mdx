Wavelet is a reactive backend. You write data in, define queries, and every subscriber (whether it's a React component or an always-on AI agent) gets live results that update automatically.

No polling. No manual cache invalidation. No WebSocket plumbing.

<CodeGroup>
```typescript Write data
await client.event('purchases').emit({
  user_id: 'alice',
  item: 'Pro Plan',
  amount: 29.99,
})
```

```typescript Subscribe to results
// React: one hook, always up to date
const { data } = useWavelet('revenue_by_user', {
  keyBy: 'user_id',
})
```

```typescript Define queries
queries: {
  revenue_by_user: {
    query: sql`
      SELECT user_id, SUM(amount) AS total
      FROM purchases
      GROUP BY user_id
    `,
  }
}
```
</CodeGroup>

## How it works

<Steps>
  <Step title="Define your data and queries">
    Write a `wavelet.config.ts` that declares your input events and the SQL queries you want to subscribe to.

    ```typescript wavelet.config.ts
    import { defineConfig, sql } from '@risingwave/wavelet'

    export default defineConfig({
      database: 'postgresql://root@localhost:4566/dev',

      events: {
        purchases: {
          columns: { user_id: 'string', item: 'string', amount: 'float' },
        },
      },

      queries: {
        revenue_by_user: {
          query: sql`
            SELECT user_id, SUM(amount) AS total, COUNT(*) AS orders
            FROM purchases GROUP BY user_id
          `,
        },
      },
    })
    ```
  </Step>
  <Step title="Start the server">
    One command sets up everything: database tables, queries, and a live API server.

    ```bash
    npx wavelet dev
    ```
  </Step>
  <Step title="Use it from your app or agent">
    Write data in. Subscribe to results. Your UI updates automatically.

    ```tsx
    import { initWavelet, useWavelet } from '@risingwave/wavelet-sdk/react'

    initWavelet({ url: 'http://localhost:8080' })

    function RevenueTable() {
      const { data, isLoading } = useWavelet('revenue_by_user', {
        keyBy: 'user_id',
      })

      if (isLoading) return <p>Loading...</p>

      return (
        <ul>
          {data.map((row) => (
            <li key={row.user_id}>
              {row.user_id}: ${row.total} ({row.orders} orders)
            </li>
          ))}
        </ul>
      )
    }
    ```

    Or let an AI agent query and act on the same data:

    ```
    Agent: "Which users spent more than $100?"
    → query({ query: "revenue_by_user", filter: {} })
    → Finds 3 users, sends them a thank-you email
    ```
  </Step>
</Steps>

## Why Wavelet

| Traditional approach | With Wavelet |
|---------------------|-------------|
| Poll your database every N seconds | Subscribers get pushed updates instantly |
| Build a WebSocket server, manage connections, handle reconnection | Built in. One line: `useWavelet('my_query')` |
| Write cache invalidation logic | No cache to invalidate, queries are always live |
| Build separate APIs for your app and your agents | Same data, same queries, same backend for apps and agents |
| Write per-tenant data filtering logic | `filterBy: 'tenant_id'` + JWT. Done. |

## Connect your existing database

Already have a Postgres database? Wavelet can subscribe to changes from it via CDC. No need to migrate your data.

```typescript
sources: {
  my_db: {
    type: 'postgres',
    connection: 'postgresql://user:pass@your-db:5432/myapp',
    tables: ['users', 'orders'],
  },
}
```

Your existing tables become available as live query sources. Combine them with Wavelet events to build queries that span both existing and new data.
