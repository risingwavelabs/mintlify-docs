You'll build a real-time leaderboard: write game scores in, subscribe to ranked results that update instantly.

## Prerequisites

- **Node.js** >= 20
- **Docker** (Wavelet starts a database automatically. Docker is the easiest way)

## Create a project

```bash
mkdir my-app && cd my-app
npm init -y
npm install @risingwave/wavelet @risingwave/wavelet-cli @risingwave/wavelet-sdk
```

```bash
npx wavelet init
```

## Define your data

Edit `wavelet.config.ts`. This is where you declare what data goes in and what queries you want to subscribe to:

```typescript wavelet.config.ts
import { defineConfig, sql } from '@risingwave/wavelet'

export default defineConfig({
  database: 'postgresql://root@localhost:4566/dev',

  // What goes in: game scores
  events: {
    game_events: {
      columns: {
        player_id: 'string',
        action: 'string',
        score: 'int',
      },
    },
  },

  // What comes out: a live leaderboard
  queries: {
    leaderboard: {
      query: sql`
        SELECT
          player_id,
          SUM(score) AS total_score,
          COUNT(*) AS games_played
        FROM game_events
        GROUP BY player_id
      `,
    },
  },
})
```

## Start the server

```bash
npx wavelet dev
```

That's it. Wavelet starts the database, creates your tables and queries, and launches a server on `http://localhost:8080`.

## Write some data

```bash
curl -X POST http://localhost:8080/v1/events/game_events \
  -H "Content-Type: application/json" \
  -d '{"player_id": "alice", "action": "win", "score": 100}'

curl -X POST http://localhost:8080/v1/events/game_events \
  -H "Content-Type: application/json" \
  -d '{"player_id": "bob", "action": "win", "score": 75}'
```

## Read the results

```bash
curl http://localhost:8080/v1/queries/leaderboard
```

```json
{
  "query": "leaderboard",
  "rows": [
    { "player_id": "alice", "total_score": 100, "games_played": 1 },
    { "player_id": "bob", "total_score": 75, "games_played": 1 }
  ]
}
```

## Subscribe to live updates

This is where it gets interesting. Open a WebSocket connection and your client gets pushed updates every time the leaderboard changes:

```javascript
const ws = new WebSocket('ws://localhost:8080/subscribe/leaderboard')

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data)
  if (msg.type === 'diff') {
    console.log('New scores:', msg.inserted)
    console.log('Updated scores:', msg.updated)
  }
}
```

Send another score in a separate terminal and watch it appear in your WebSocket client instantly:

```bash
curl -X POST http://localhost:8080/v1/events/game_events \
  -H "Content-Type: application/json" \
  -d '{"player_id": "alice", "action": "win", "score": 200}'
```

## Use it in React

The SDK gives you a single hook that handles everything: initial fetch, WebSocket subscription, and state management:

```tsx
import { initWavelet, useWavelet } from '@risingwave/wavelet-sdk/react'

initWavelet({ url: 'http://localhost:8080' })

function Leaderboard() {
  const { data, isLoading } = useWavelet('leaderboard', {
    keyBy: 'player_id',
  })

  if (isLoading) return <p>Loading...</p>

  return (
    <ul>
      {data.map((row) => (
        <li key={row.player_id}>
          {row.player_id}: {row.total_score} pts ({row.games_played} games)
        </li>
      ))}
    </ul>
  )
}
```

Your component re-renders automatically when the leaderboard changes. No polling, no manual state updates.

## Generate typed clients

Want full type safety? One command generates typed interfaces from your config:

```bash
npx wavelet generate
```

Now your queries and events are fully typed: autocomplete, compile-time checks, the works.

## Deploy

When you're ready for production, `wavelet push` syncs your config to any database:

```bash
npx wavelet push
```

It only applies what changed. Safe to run multiple times.
